Boot Order — autostart.sh
Run: bash ~/autostart.sh — single command that starts everything.
1️⃣Kill All Processes
$pkill -f openclaw-gateway || true $pkill -f oc_brain || true
All pkills have || true — safe with set -euo pipefail
2️⃣Sync Providers from Master
$python3 -c "sync providers + channels + tools from master → openclaw.json"
Restores API keys and security from locked master
Does NOT restore primary/fallbacks — select-primary handles that
3️⃣Start Gateway
$nohup openclaw gateway run > /dev/null 2>&1 &
Uses "run" — required for Termux environment
Waits in loop up to 60s until port 18789 responds
4️⃣Wait for Gateway Ready
$for i in $(seq 1 30); do curl -s http://127.0.0.1:18789/health | grep -q "ok" && break; sleep 2; done
Polls every 2s — gateway ready in ~22 seconds
Brain only starts AFTER this
5️⃣Select Primary Model
$timeout 30 python3 ~/oc_brain.py --select-primary || true
Reads brain_state.json — skips keys in 429 cooldown
Tests fallbacks in order — picks first working key
30s timeout — never blocks boot
6️⃣Start Brain + Scraper + Dashboard
$nohup python3 ~/oc_brain.py > /dev/null 2>&1 & $cd ~/scraper && bash start_dashboards.sh || true $nohup python3 ~/dashboard_server.py > /dev/null 2>&1 &
7️⃣Tunnel (On Demand Only)
$bash ~/tunnel_keeper.sh
NOT in autostart — run manually when needed
Sends public URL to Telegram automatically
